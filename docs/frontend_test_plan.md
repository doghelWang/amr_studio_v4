# AMR Studio V4 前端交互与配置总成测试方案

## 1. 测试目标
全面验证 AMR Studio V4 前端工程在配置交互、状态管理（Zustand + Zundo）、实时校验（Validation Engine）以及项目持久化方面的功能完整性与稳定性。

## 2. 测试环境
- **前端地址**: `http://localhost:3000`
- **后台接口**: `http://localhost:8000/api/v1`

## 3. 测试用例矩阵

### TC01: 基础架构与持久化
- **步骤**:
  1. 刷新页面，确认初始状态为“未命名项目”，且配置表单为默认值。
  2. 修改“基础信息”中的“项目名称”为 `Test_Project_01`。
  3. 点击“保存”图标下载 `.amrproj` 文件。
  4. 刷新页面恢复初始状态，然后点击“打开”图标，加载下载的 `.amrproj`。
- **预期结果**: 项目名称和所属数据能够完全恢复。

### TC02: 状态管理与时空穿越 (Undo/Redo)
- **步骤**:
  1. 切换至“轮组驱动”面板。
  2. 将左轮组的“最大速度(Idle)”修改为 `3000`。
  3. 点击顶部工具栏的“撤销 (Undo)”。
  4. 点击顶部工具栏的“重做 (Redo)”。
- **预期结果**: 撤销后值恢复为初始值（如 `1500`），重做后恢复为 `3000`。多次切换标签页不影响撤销栈。

### TC03: 复杂校验规则拦截
- **步骤**:
  1. 切换至“主控配置网络与总线”面板。
  2. 观察当前 CAN Bus 列表（例如有 CAN0, CAN1）。
  3. 切换至“轮组驱动”面板，将右轮（Right Drive Motor）的 CAN Node ID 改为 `1`（与左轮冲突）。
  4. 观察右上角“Health Badge”健康度指示灯。
  5. 点击指示灯打开健康度面板。
- **预期结果**: 指示灯变红并提示 `1` 个报错。健康度面板明确指出“ID=1 冲突”。

### TC04: 动力拓扑安全切换与数据重置
- **步骤**:
  1. 切换至“基础信息”面板。
  2. 将驱动拓扑从 `差速双驱 DIFFERENTIAL` 切换为 `四舵轮 QUAD_STEER`。
  3. 在弹出的二次确认对话框中，点击“取消”。
  4. 再次切换为 `四舵轮 QUAD_STEER`，并点击“直接切换（丢弃现有配置）”。
  5. 切换回“轮组驱动”面板。
- **预期结果**: 点击“取消”时不发生任何改变。点击“直接切换”后，轮组驱动面板中应该出现 4 个轮组卡片（前左、前右、后左、后右）。

### TC05: 分段控制器（空载/满载）联动
- **步骤**:
  1. 在“轮组驱动”面板的任意轮组中，观察“安装位置”和“运动参数”的 Segmented 按钮。
  2. 在“位置”一栏点击“满载”，将 Head Offset 改为 `2000`。
  3. 切换至“电气接线”再切换回“安装位置”。
  4. 切换为“空载”。
- **预期结果**: UI 应正确隔离并持久化空载与满载的数据，切换标签页或轮组时，状态独立不粘连。

### TC06: 编译与生成验证
- **步骤**:
  1. 在没有任何报错（Health Badge 绿色）的情况下，点击顶部工具栏的“编译”大按钮。
- **预期结果**: 系统提示编译成功，并触发浏览器下载对应的 `Test_Project_01_ModelSet.zip` 成果物压缩包。
