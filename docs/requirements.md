## 📡 GitHub 命令桥接协议 (Interaction Guide)
为了实现高效的异步协作，请遵循以下规则与 AI 团队沟通：

1. **下达指令**：在 GitHub 网页端直接编辑本文件，在最下方的【反馈区】添加新的 `## 📥 FEEDBACK_V[编号]`。
2. **AI 响应**：哨兵进程每 60 秒扫描一次更新。一旦检测到变化，将自动执行：
   - **分析指令** -> **修改代码** -> **重新构建 .cmodel** -> **推送至 GitHub**。
3. **查看回执**：请关注同目录下的 `TEAM_LATEST_STATUS.md`，它会显示当前处理进度。
4. **验证成果**：AI 推送完成后（Commit 带有 `[AI-REPLY]`），请下载 `gemini_audits/` 下的模型进行验证。

## 📅 [2026-03-09] - P1: 基于业务规则的可视化强校验反馈
**状态**: 🔴 待处理

### 需求描述
当前系统虽然在底层加入了 Zod 等数据校验，但用户在填写大批量表单数据（如 CAN ID 冲突、未分配 IP 等）时，缺乏足够直观的前端界面反馈。我们需要增强 UI 层的验证提示。

### 验收标准 (Acceptance Criteria)
- [ ] 所有配置项需封装为 `ValidatedInput` 组件，实现输入时（onChange/onBlur）的实时行内红字错误提示。
- [ ] 左侧的安装蓝图与电气接线图 (Canvas) 若存在定位重叠或未接线的异常组件，需在图表上自动高亮或显示红色警告外框。

## 📅 [2026-03-09] - P2: 原生拖拽级画布深度编辑
**状态**: 🔴 待处理

### 需求描述
当前 2D 安装蓝图 (RobotCanvas) 和接线图 (WiringCanvas) 已经实现了与 Store 的单向绑定显示。为了提高工程效率，用户希望可以直接在画布上用鼠标拖拽来改变模型坐标。

### 验收标准 (Acceptance Criteria)
- [ ] 实现在安装蓝图上拖动传感器图标或驱动轮图标时，自动解算 `X/Y` 坐标并回写到 `useProjectStore` 中。
- [ ] 确保拖拽结束后 (`onDragEnd`)，左侧配置列表的数字输入框同步瞬间变动。
- [ ] 在较大的底盘可视化时，提供画布的滚轮缩放 (Zoom) 和平移 (Pan) 大盘视角功能。

## 📅 [2026-03-09] - P3: 项目导出与一键报告生成体系
**状态**: 🔴 待处理

### 需求描述
除了直接输出运行机器人的 `.model` 二进制文件，企业还需要用于存档的研发与生产制造文档。需要能一键导出清单和报告。

### 验收标准 (Acceptance Criteria)
- [ ] 新增一个 PDF Compilation API：能够把当前的设备清单、模型长宽、组装图纸自动排版并生成 PDF 研发说明书。
- [ ] 新增 BOM (Bill of Materials) 导出：自动统计算出需要多少个电机、几条 CAN 总线线束、几个传感器，导出为标准 `.csv` 文件供采购部使用。
