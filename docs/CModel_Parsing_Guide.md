# CModel 解析指南 (V7 标准)

CModel 是 AMR Studio V4 用于描述机器人完整硬件树和电气网络架构的核心文件格式。本质上它是一个序列化容器。以下是逆向解析其结构的完整指南，这些结论已被 `parse_v7.py` 100% 验证成功。

## 1. 物理封装与解压
一个 `.cmodel` 文件实际上是一个标准的 ZIP 压缩包。
里面包含两个主要文件：
1. `baseInfo.json`：包含该工程的基本信息描述（项目名、时间、版本等），为纯文本 JSON，易于解析。
2. `.model` 文件（通常以工程名命名）：这是包含了 AMR 物理结构和拓扑关系的 **Protobuf 二进制序列化文件**。

## 2. 二进制拓扑反序列化
`.model` 文件没有提供内嵌结构描述（即没有 `.proto` 的文本），因此我们必须依赖系统预先下发的 Schema 定义。
核心要点：
- **目标 Schema**：解析需使用 `CompDesc.model` 定义的结构（Message Type 为 `amr_cmodel.comp_desc_list`）。
- **工具链**：使用 `blackboxprotobuf` 作为反序列化引擎，将二进制无损转化为 Python 的嵌套字典。
- **编码处理**：此时字典内部的所有字符串皆为 `bytes` 或 `bytearray`。需要编写一个**深度递归**的 `decode_str` 函数，逐层遍历字典/列表，尝试使用 `utf-8` 或 `gbk` 剥离出正确的 Unicode 中文字符和英文标识。

## 3. 核心节点：模块列表 `[Tag 5]`
反序列化出来的大字典 `D` 中，`D['5']` 是一个数组，内部包含了机器人所有的模块节点。我们将其内部每个元素称为 `Node`。
`Node` 包含以下直观标签：
- **`5`**：模块的自身 UUID
- **`6`**：模块类型（如 `chassis`, `driver`, `camera`）
- **`7`**：模块实例名称（如 `driver-left`, `chassis_diff`）
- **`8`**：模块物理尺寸（即 `LxWxH` 字符串）

真正的核心深层配置隐藏在 `Node['4']`（一个数组，通常其首元素 `m = Node['4'][0]` 包含了所有细节）。

## 4. 深层结构解析 `m = Node['4'][0]`

### 4.1 空间位置与归属 `m['1']` (Relations)
该标签下为一个数组，每个元素描述了该模块相对于系统或其他模块的空间姿态或链接从属。
- **`4`**：目标（父）UUID
- **`5`**：标记类型（如果值为 `P`，代表本模块是子级，`4` 中的 UUID 是其父级模块）
- **空间姿态 `6` ~ `11`**：X / Y / Z 的平移，以及 Roll / Yaw / Pitch 的旋转度数（通常使用 IEEE 754 浮点数表示）。使用 `blackboxprotobuf` 取出即可。

### 4.2 模块私有电气与机械属性 `m['2']` (Private Attributes)
这里存储了模块出厂的私有属性，如底盘的加速度、偏航角、软件规格等。
- **`1`**：属性群组结构。根据节点内部声明组的数量，这里可能是一个字典（只有一组时）或数组（包含多个组结构）。**必须对这种单/复数二相性做鲁棒兼容。**
- **组名**：从 `group['2']` 中提取，例如“运动中心参数”、“底盘参数”等。
- **独立属性集合**：从 `s = group['3']` 这个数组中遍历取出。
  - `s['51']` 代表这句属性的字段名。
  - 具体的值可能会被储存在不同类型的标签（如 `17`, `35`, `45`, `10` 等，需要通过动态探测字典的主键取出对应值）。

### 4.3 接口与总线拓扑属性 `m['3']` (Interfaces)
极其核心的字典结构，用于表示机器人的对外接头和总线连接。
- 根级同样是一个数组，每一个元素的 `1` 键代表接口总称（如 `CAN`, `DI`, `ETH`）。
- 该元素的 `3` 键为一个数组或字典，包含了这个总接口下的分支端口。
- 针对单个分支端口 `port`：
  - **`1`**：端口具体名称（如 `CAN_1`, `ETH_2`）。
  - **`5`**：该端口独立的自身 UUID。
  - **`8` 与 `9`**：分别是**硬件属性**和**软件属性**。其内部的 `3` 都会指向一个属性列表（如同私有属性一样，包含 `51`=参数名，其他标签=参数值的数据对，诸如波特率、ip参数等均在这里）。
  - **`6`**：【最核心】**电气线路连接指针**。该键的值是一个字符串数组，储存着与它连接的其他模块上的**端口的 UUID**。

## 5. 电气接线图构建算法 (Pass 1 & Pass 2)
由于文件中并不是存储“谁连接到了 driver-left_CAN1”，而是存储“谁连上了 UUID_XYZ”，因此我们必须两轮处理（2-Pass）解开拓扑关系。

- **Pass 1 (端口注册)**：在遍历所所有模块结构时，建立一个全局词典 `uuid_to_port`。当遇到任何一个分支端口 `port` 时，读取其 `port['5']` 作为键，将 `{module_name}, {port_name}` 作为值记录。
- **Pass 2 (寻址映射)**：解析完毕后，遍历所有之前遇到过 `port['6']`（目标连接UUID列表）的端口。对其中的每一个目标 UUID，去全局词典 `uuid_to_port` 中查找它指向哪个模块的哪个名称。借此瞬间生成 `driver-left CAN_1 <---> MainController CAN_2` 的精准连线表。

这就是完整的 100% 精度解析流程。在未来的重构中，需要将其完整迁移作为生成端（序列化器）的逆向操作参照物。
